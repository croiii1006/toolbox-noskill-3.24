# ORANGEN 流程与内容梳理

## 1. 模块定位

ORANGEN 是当前项目里的内容生成工作台，对应前端模块为 `skills`。

它的目标不是单纯“生成一条视频”，而是把以下输入逐步转成一条可干预、可恢复、可追踪的爆款视频生成链路：

- 商品图
- 品类
- 卖点
- 记忆库资料
- 爆款参考视频

最终产物包括：

- 候选爆款视频
- 反向拆解出的 Prompt
- 用户确认后的最终生成结果

当前实现以前端流程编排和交互验证为主，很多执行过程仍然是模拟态，用来表现多 Agent 协作和任务推进。

## 2. 页面总结构

ORANGEN 页面可以理解成一个“双栏工作台”。

### 2.1 左侧

左侧是主流程区，负责承接用户操作和流程推进。它会随着状态切换展示不同内容：

- 首页输入态
- 已启动后的消息流
- Setup 摘要
- 流程步骤卡片
- Agent 创建信息
- 视频选择确认信息
- 生成状态信息

### 2.2 右侧

右侧是工作区，负责展示结构化产物和细节操作，主要包括：

- Checklist
- 各 Agent 详情
- 记忆库阅读面板
- Prompt 编辑器
- 结果预览区

左侧更像“流程时间线”，右侧更像“当前阶段的操作台”。

## 3. 入口与输入

### 3.1 默认入口

用户从侧边栏进入 `ORANGEN` 首页后，会先看到一个极简输入界面。

当前首页需要用户补齐的核心输入包括：

- 商品白底图
- 品类
- 卖点标签
- 可选记忆库附件

### 3.2 来自 ORANHUB 的预填

ORANGEN 支持从 `ORANHUB` 直接带上下文跳入。

当前已支持透传：

- 洞察报告附件
- 策划方案附件
- 品类

透传规则是“有什么传什么”：

- 只有洞察，就只带洞察
- 只有策划，就只带策划
- 两者都有，就一起作为记忆库附件带入

进入 ORANGEN 后：

- 品类会自动预选
- 对应记忆库条目会自动预选

## 4. 左侧页面状态与 UI 逐步梳理

这一部分重点说明“左侧每一个流程怎么走、用户看到什么 UI、点了会发生什么”。

### 4.1 状态 A：首页输入态

这是用户刚进入 ORANGEN 时的默认状态。

#### 左侧 UI

左侧页面会显示：

- 顶部大标题 `ORANGEN`
- 一句副标题，说明可以上传商品图或直接开始对话
- 一个居中的输入卡片

输入卡片内部包含：

- 商品图上传框
- 品类选择器
- 卖点标签输入区
- 记忆库按钮
- 预计消耗提示
- 右侧提交按钮

如果有来自 ORANHUB 的透传：

- 品类会自动填上
- 记忆库会自动带入对应条目

#### 用户操作

用户在这个阶段主要完成 4 件事：

1. 上传商品图
2. 选择品类
3. 输入卖点并回车生成标签
4. 选择记忆库资料

#### 进入下一步的条件

当基础输入满足条件后，用户点击提交按钮，系统会创建一次新的会话 Setup，并正式启动整条生成流程。

### 4.2 状态 B：会话启动后，左侧进入消息流模式

一旦用户正式提交，左侧页面会从“首页输入态”切换成“流程消息流态”。

#### 左侧 UI

左侧会变成一个类似对话流的工作区，顶部和中部结构如下：

- 顶部栏
  - 左上角有“返回”按钮
  - 用于退出当前会话并回到初始输入页
- 中部消息流区域
  - 展示 Setup 摘要
  - 展示流程步骤卡片
  - 展示系统状态文本
  - 展示 Agent 创建和运行信息

这个阶段左侧不再展示首页大标题，而是切换成“流程进行中”的布局。

### 4.3 状态 C：Setup 摘要展示

在流程正式开始后，左侧会先插入一块 Setup 摘要，帮助用户确认本次任务的输入基础。

#### 左侧 UI

这一块通常会概括展示：

- 当前品类
- 当前卖点
- 是否启用了记忆库
- 已选择的记忆库条目
- 已上传的商品图信息

#### 作用

这一步的意义是把“用户原始输入”固化成一份任务起点，让后续流程都围绕这份 Setup 展开。

### 4.4 状态 D：流程步骤卡片出现

接下来左侧会出现一个流程步骤卡片，它不是简单文字，而是一个聚合型的流程面板。

#### 左侧 UI

这类卡片里会逐条出现：

- 编写待办清单
- 创建助手
- 阅读待办清单
- 阅读记忆库
- Agent 分组信息

在视觉上，这些步骤会按顺序堆叠，并通过虚线连接，形成一条“工作流轨迹”。

#### 交互方式

其中部分步骤是可点击的：

- 点待办清单，会打开右侧 Checklist
- 点阅读记忆库，会打开右侧记忆库阅读面板

也就是说，左侧这里不是纯展示，它承担“流程导航入口”的作用。

### 4.5 状态 E：Agent 创建与说明阶段

在左侧流程卡片里，系统会逐步展示各类 Agent 的创建过程。

#### 左侧 UI

这部分通常表现为：

- “创建助手”步骤
- 下面展开一个 Agent 列表
- 每个 Agent 一行
- 行内显示：
  - 创建图标
  - Agent 名称
  - 展开箭头

展开后还能看到该 Agent 的任务背景说明。

#### 当前 Agent 分工

目前系统内的 Agent 主要有：

- Agent 01：TikTok 爆款专家
- Agent 02：记忆库专家
- Agent 03：Prompt 专家
- Agent 04：视频生成专家

#### 作用

这一阶段在左侧承担的是“告诉用户现在有哪些专家被拉进来一起工作”。

### 4.6 状态 F：候选视频生成完成前的运行提示

当系统开始运行 Agent01 时，左侧会继续插入消息和状态信息。

#### 左侧 UI

用户会看到一系列流式文本，例如：

- 开始抓取
- 正在分析匹配度
- 正在排序
- 正在提取封面

这些消息有些会以普通文本出现，有些会配上图标，形成“系统正在推进”的感受。

#### 作用

它的主要功能是降低等待空白感，让用户知道系统不是卡住，而是在执行任务。

### 4.7 状态 G：候选视频已生成，等待用户选视频

当 Agent01 结束后，左侧不会直接给出最终结果，而是进入“等待用户决策”的状态。

#### 左侧 UI

左侧这时会出现一条确认性质的文本，例如说明候选已经准备好。

同时右侧会切到 Agent01 工作区，展示候选视频列表。

#### 用户动作

用户需要去右侧做一个关键动作：

- 从候选视频中选中一个参考爆款视频

#### 左侧的作用

左侧在这个阶段主要起承接作用：

- 告诉用户流程走到了哪里
- 引导用户去右侧完成下一步选择

### 4.8 状态 H：用户确认了参考视频

当用户在右侧选中一个候选视频后，左侧会插入一条“选择已确认”的消息。

#### 左侧 UI

这条消息通常表现为：

- 一条确认型状态文案
- 用于说明当前参考视频已经确定

#### 作用

这表示流程正式从“找参考”进入“做拆解”。

### 4.9 状态 I：记忆库构建阶段

如果这次任务启用了记忆库，左侧流程会继续进入记忆增强阶段。

#### 左侧 UI

此时左侧会继续出现流程卡片或状态文案，说明系统正在：

- 连接记忆库
- 检索相关记忆
- 构建上下文向量

如果没有启用记忆库：

- 这部分会被跳过或弱化
- 左侧仍然会继续推进到 Prompt 阶段

#### 作用

这一步的意义是告诉用户：

- 现在不是单纯照抄爆款视频
- 系统会把品牌 / 产品 / 既有报告上下文一起注入后续生成

### 4.10 状态 J：Prompt 生成阶段

当视频样本和记忆库上下文都就位后，系统进入 Agent03 阶段。

#### 左侧 UI

左侧会继续输出日志式的推进信息，例如：

- 开始分析视频内容
- 提取关键帧
- 提取风格特征
- 生成 Prompt

#### 左侧的作用

这一阶段左侧负责告诉用户：

- 系统正在把“视频参考 + 记忆上下文”转成一份结构化 Prompt

真正的编辑动作则发生在右侧。

### 4.11 状态 K：Prompt 已生成，等待用户确认

这是左侧流程中的一个关键停顿点。

#### 左侧 UI

左侧会出现一条“Prompt 已生成”或类似状态消息，说明系统已完成自动拆解。

同时右侧会打开 Agent03 面板，展示：

- 工作日志
- Prompt 编辑器
- “确认并生成”按钮

#### 用户动作

用户此时需要去右侧完成：

- 查看 Prompt
- 手动调整 Prompt
- 点击“确认并生成”

#### 左侧在这里的角色

左侧负责记录这个阶段已经完成，但不会承载编辑器本身。

### 4.12 状态 L：进入最终生成阶段

当用户在右侧确认 Prompt 后，左侧会再次进入运行提示态。

#### 左侧 UI

用户会在左侧看到新的生成状态文本，例如：

- 开始渲染
- 合成音频
- 组合视频
- 正在输出结果

#### 作用

它的作用和前面的运行提示一致：

- 持续反馈
- 告知当前阶段已经从“拆解”切换成“生成”

### 4.13 状态 M：最终结果完成

当 Agent04 完成后，整个流程进入完成态。

#### 左侧 UI

左侧会出现完成型消息，说明本次流程已经结束。

这时右侧会展示结果预览区。

#### 用户可以做什么

用户后续通常会：

- 查看结果
- 重新生成
- 结束本次会话

## 5. 左侧 UI 组件拆解

为了便于研发或产品对齐，下面把左侧的 UI 组成再拆一层。

### 5.1 顶部操作区

在首页输入态：

- 右上角有历史记录入口

在流程进行态：

- 左上角有返回按钮
- 顶部不再保留首页标题

### 5.2 消息流区域

左侧核心区域是一个可滚动消息流。

它可以承载以下类型内容：

- Setup 摘要卡
- 流程分组卡
- 选择确认消息
- 普通文本消息
- 生成状态消息

### 5.3 流程分组卡

这块是左侧最有结构感的 UI。

它通常由以下元素组成：

- 图标
- 步骤标签
- 连接虚线
- 可点击的跳转项
- Agent 子步骤

### 5.4 Agent 展示区

Agent 在左侧并不是大卡片，而更像嵌在流程里的“创建与说明模块”。

单个 Agent 条目通常包括：

- 头像或图标
- Agent 名称
- 展开按钮
- 展开后的任务背景说明

### 5.5 状态消息

左侧还会穿插很多状态型消息。

这些消息用于表达：

- 已开始
- 已确认
- 处理中
- 已完成
- 异常

它们让整个页面像一条被记录下来的执行轨迹，而不是静态表单。

## 6. 右侧工作区结构

虽然这份文档重点在左侧，但右侧仍需要简要说明，因为很多左侧步骤会把用户导向右侧。

### 6.1 Checklist 视图

展示待办清单和完成状态。

### 6.2 Agent01 视图

展示：

- 工作日志
- 候选爆款视频列表

用户需要在这里选择参考视频。

### 6.3 Agent02 视图

展示：

- 记忆库处理日志

如果没有启用记忆库，会显示未启用状态。

### 6.4 Agent03 视图

展示：

- 工作日志
- Prompt 编辑器
- 返回重新选视频
- 确认并生成

### 6.5 Agent04 视图

展示：

- 生成日志
- 结果预览

### 6.6 记忆库阅读视图

当左侧点击“阅读记忆库”时，右侧会切成文档阅读模式。

阅读面板会显示：

- 记忆库标题
- 类 markdown 的内容区
- 行号
- 复制按钮

## 7. 用户交互关键节点

从体验上看，ORANGEN 有 3 个关键停顿点。

### 7.1 启动任务

用户补齐首页输入后，正式开始任务。

### 7.2 选择参考视频

这是从“信息收集”切到“内容拆解”的关键点。

### 7.3 确认 Prompt

这是从“拆解思路”切到“正式生成”的关键点。

因此 ORANGEN 不是一次性黑盒生成，而是一条带人工确认节点的半自动工作流。

## 8. 历史记录机制

ORANGEN 已接入历史记录，用于保存和恢复会话。

历史中会保存：

- Setup
- 当前任务状态
- 候选视频
- 已选视频
- Prompt
- 最终结果

历史支持：

- 恢复会话
- 删除单条记录
- 识别进行中的任务

所以用户可以从中途中断的地方继续，而不是每次重头开始。

## 9. 与 ORANHUB / ORAN SIM 的关系

### 9.1 与 ORANHUB

ORANHUB 负责洞察和策划产出，ORANGEN 负责把这些上游结论进一步用于内容生成。

当前已打通：

- ORANHUB 可把洞察报告 / 策划方案写入记忆库
- ORANHUB 可带着品类与附件跳入 ORANGEN
- ORANGEN 会把这些资料作为记忆上下文参与内容生成

### 9.2 与 ORAN SIM

两者都能使用记忆库，但目标不同：

- ORAN SIM：做预测 / 仿真 / 推演
- ORANGEN：做生成 / 复刻 / 输出内容

## 10. 当前实现状态说明

从代码实现看，ORANGEN 当前更接近一个高保真流程原型。

已完成的部分：

- 多阶段工作流结构
- 左右分栏工作台
- Agent 分工与状态推进
- 候选视频选择
- Prompt 编辑与确认
- 历史记录
- 来自 ORANHUB 的附件与品类透传

仍待真实接入的部分：

- TikTok 抓取与排序的真实后端能力
- 记忆库检索与向量构建服务
- Prompt 生成模型的真实调用
- 最终视频合成与真实结果返回

## 11. 一句话总结

ORANGEN 是一个“带记忆库上下文的爆款内容生成工作流”，左侧负责把用户操作和流程推进记录成一条清晰的执行轨迹，右侧负责承接每个关键阶段的细节操作与结构化结果。
